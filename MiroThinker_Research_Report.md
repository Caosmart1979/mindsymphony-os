## 📋 执行摘要

**MiroThinker** 是一个开源的研究智能体套件，专注于通过深度推理实现长上下文理解。该项目提供了一系列开箱即用的智能体，无需向量数据库即可实现高质量的检索增强生成（RAG）。

### 核心亮点
- 🧠 **深度推理**: 基于思维链的长文本理解
- 🚀 **零依赖 RAG**: 无需向量数据库的原生实现
- 🤖 **多智能体**: 7+ 开箱即用的研究智能体
- 📚 **长上下文**: 支持超长文本处理

## 🔧 核心组件

### 1. MiroEngine - 推理引擎

**功能**: 深度推理的核心引擎

```python
# 伪代码示例
class MiroEngine:
    def __init__(self):
        self.thought_chain = ThoughtChain()
        self.context_manager = ContextManager()

    def reason(self, query, documents):
        # 1. 理解查询
        intent = self.understand(query)

        # 2. 构建思维链
        chain = self.thought_chain.build(intent, documents)

        # 3. 深度推理
        result = self.deep_reasoning(chain)

        # 4. 优化输出
        return self.optimize(result)
```

**特点**:
- 支持多步推理
- 可解释的思维链
- 自适应推理深度

### 2. ThoughtChain - 思维链框架

**功能**: 结构化思维过程

```yaml
thought_chain:
  steps:
    - step: "问题分解"
      action: "将复杂问题拆解为子问题"

    - step: "信息检索"
      action: "从文档中提取相关信息"

    - step: "逻辑推理"
      action: "基于提取信息进行推理"

    - step: "答案综合"
      action: "整合推理结果生成答案"
```

### 3. ContextManager - 上下文管理器

**功能**: 管理长上下文

**特点**:
- 智能上下文窗口
- 动态上下文压缩
- 上下文优先级排序

### 4. RAGOptimizer - RAG 优化器

**功能**: 无向量数据库的 RAG 实现

**原理**:
```
传统 RAG:
文档 → 向量化 → 存储 → 检索 → 生成
         ↓
    向量数据库 (成本高)

MiroThinker RAG:
文档 → 索引 → 智能匹配 → 深度推理 → 生成
         ↓
    原生索引 (零成本)
```

## 🚀 快速开始

### 安装

```bash
# 克隆项目
git clone https://github.com/MiroMindAI/MiroThinker.git

# 安装依赖
cd MiroThinker
pip install -r requirements.txt
```

### 基本使用

```python
from mirothinker import MiroEngine

# 初始化引擎
engine = MiroEngine()

# 加载文档
documents = engine.load_documents("path/to/docs")

# 查询
result = engine.query(
    "这个研究的主要发现是什么？",
    documents=documents
)

# 输出
print(result.answer)
print(result.reasoning_chain)
```

### 使用内置智能体

```python
from mirothinker.agents import MiroSummarizer

# 初始化摘要智能体
summarizer = MiroSummarizer()

# 生成摘要
summary = summarizer.summarize(
    document="long_research_paper.pdf",
    style="academic",
    length="medium"
)
```
