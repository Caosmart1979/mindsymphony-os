使用Gemini CLI处理视频的实战示例。

## 场景1: Bug复现视频分析

### 触发语

```
Gemini帮我分析这个Bug复现视频，找出问题根因 bug-repro.mp4
```

### 执行命令

```bash
gemini -p "这是一个Bug复现视频。请：

## 分析要求
1. **时间线梳理**: 列出视频中每个关键操作和时间点
2. **问题定位**: 准确指出Bug出现的时刻和现象
3. **上下文分析**: Bug发生前的操作序列
4. **根因推测**: 基于视频内容推测可能的技术原因

## 输出格式
### 视频概要
- 时长: 
- 主要内容:

### 操作时间线
| 时间 | 操作 | 结果 |
|------|------|------|

### Bug分析
- 出现时刻:
- 现象描述:
- 推测原因:
- 建议排查:" bug-repro.mp4 --yolo
```

### 预期输出

```markdown
### 视频概要
- 时长: 18秒
- 主要内容: 用户在表单提交时遇到重复请求问题

### 操作时间线
| 时间 | 操作 | 结果 |
|------|------|------|
| 0:00-0:03 | 填写表单字段 | 正常 |
| 0:03 | 点击提交按钮 | loading出现 |
| 0:05 | loading消失 | 页面无响应 |
| 0:07 | 再次点击提交 | 第二个loading出现 |
| 0:10 | 两个请求同时返回 | 显示错误toast |
| 0:12-0:18 | 页面卡顿 | 控制台报错 |

### Bug分析
- **出现时刻**: 0:05，第一次请求后loading消失但按钮未禁用
- **现象描述**: 按钮可重复点击导致多次提交
- **推测原因**: 
  1. 请求状态管理问题
  2. 按钮loading状态未正确绑定
  3. 可能缺少防抖/节流
- **建议排查**:
  1. 检查isLoading状态与按钮disabled的绑定
  2. 添加请求去重逻辑
  3. 考虑乐观更新策略
```

## 视频预处理技巧

### 压缩大视频

```bash
# 降低分辨率
ffmpeg -i large.mp4 -vf scale=720:-1 -c:a copy smaller.mp4

# 降低码率
ffmpeg -i large.mp4 -b:v 1M -b:a 128k compressed.mp4
```

### 截取片段

```bash
# 截取前2分钟
ffmpeg -i long.mp4 -ss 00:00:00 -t 00:02:00 -c copy clip.mp4

# 截取特定时间段
ffmpeg -i long.mp4 -ss 00:05:30 -to 00:08:00 -c copy segment.mp4
```

### 提取音频（如只需语音内容）

```bash
ffmpeg -i video.mp4 -vn -acodec mp3 audio.mp3
```

## 支持的视频格式

| 格式 | 支持状态 | 备注 |
|------|----------|------|
| MP4 | ✅ 推荐 | 最佳兼容性 |
| MOV | ✅ | Apple格式 |
| AVI | ✅ | 较老格式 |
| MKV | ⚠️ | 可能需转换 |
| WebM | ✅ | Web格式 |

---

## 注意事项

1. **视频时长**: 过长视频建议切分或提取关键片段
2. **清晰度**: 确保视频足够清晰，尤其是包含文字/代码时
3. **音频质量**: 如需转录，确保音频清晰
4. **敏感内容**: 检查视频中是否包含敏感信息
5. **文件大小**: 大文件上传可能耗时较长
