## 场景1: 图片分析

### 适用情况

- UI截图审查
- 设计稿分析
- 错误截图诊断
- 图表数据提取

### 使用方式

```
用Gemini分析这张截图 ~/Screenshots/bug.png
```

### Gemini执行的命令

```bash
gemini -p "请详细分析这张图片：
1. 图片内容描述
2. 如果是UI，分析布局和交互元素
3. 如果有文字，提取关键信息
4. 发现的问题或改进建议" \
  ~/Screenshots/bug.png --yolo
```

### 实际案例

**场景**: 前端同事发来一张移动端弹窗截图，说"圆角和阴影不对"

**你的输入**:
```
用Gemini帮我分析这个UI截图有什么问题 popup-design.png
```

**Gemini返回**:
```
分析结果：

1. 圆角问题
   - 当前弹窗圆角约4px，视觉偏硬
   - 建议改为12-16px以符合现代设计趋势

2. 阴影问题
   - 当前阴影过淡(约5%透明度)，层次感不足
   - 建议: box-shadow: 0 4px 20px rgba(0,0,0,0.15)

3. 其他发现
   - 按钮间距不一致（左16px，右12px）
   - 关闭图标略小，建议24x24px
```

```bash

1. 视频主要内容
2. 关键操作步骤和时间点

  ~/Videos/bug-demo.mp4 --yolo
```

### 实际案例

**场景**: QA发来15秒的Bug复现录屏

**你的输入**:
```
Gemini帮我看看这个Bug视频，找出问题在哪 bug-repro.mp4
```

```
Bug分析：

时间线：
- 0:00-0:03 用户点击"提交"按钮
- 0:03-0:05 loading动画显示
- 0:05-0:08 loading消失，但页面无响应

- 请求期间禁用按钮
```

## 场景5: 代码仓库扫描

### 适用情况

- 新项目接手
- 架构评审
- 代码质量分析
- 安全漏洞扫描

### 使用方式

```
用Gemini扫描整个项目，分析架构
```

### Gemini执行的命令

```bash
cd /path/to/project && gemini --all-files -p "请分析这个代码仓库：
1. 项目结构和模块划分
2. 技术栈和框架
3. 核心业务逻辑
4. 潜在问题和改进建议" --yolo
```

### 实际案例

**场景**: 接手一个遗留项目

**你的输入**:
```
Gemini帮我分析这个项目的架构，我需要了解整体结构 /path/to/legacy-project
```

**Gemini返回**:
```
项目分析：legacy-project

技术栈：
- 后端：Node.js + Express + MongoDB
- 前端：React 16 + Redux
- 构建：Webpack 4

目录结构：
├── src/
│   ├── api/          # REST接口(47个端点)
│   ├── models/       # 数据模型(12个)
│   ├── services/     # 业务逻辑(8个服务)
│   └── utils/        # 工具函数
├── client/           # 前端代码
└── tests/            # 测试(覆盖率约35%)

核心模块：
1. 用户认证 - JWT方案，有刷新token
2. 订单系统 - 状态机设计，6个状态
3. 支付集成 - 支持支付宝/微信

发现的问题：
⚠️ 高优先级：
- 无请求限流，存在DDoS风险
- 部分SQL拼接，有注入风险
- 密码存储用MD5，应升级bcrypt

📝 中优先级：
- 前端React版本过旧
- 缺少错误边界处理
- 日志不规范

💡 改进建议：
1. 立即修复安全问题
2. 添加请求限流中间件
3. 规划前端框架升级
```

## 场景组合：多模态工作流

### 示例：完整的Bug处理流程

```
1. 收到Bug截图
   → "Gemini分析这张Bug截图 bug.png"

2. 收到复现视频
   → "Gemini看这个复现视频，找出问题 bug.mp4"

3. 需要改代码
   → Claude Code主导修复

4. 需要更新文档
   → Claude Code编写文档

5. 提交前审查
   → "Gemini扫描改动的文件，检查有没有遗漏"
```

整个流程中：
- 多模态任务 → Gemini处理
- 编码任务 → Claude Code处理
- 主对话保持清爽
- 各司其职，效率最大化

## 最佳实践

### DO ✅

1. **明确指定文件路径**：让Gemini知道处理什么
2. **说明期望输出**：想要摘要、列表还是详细分析
3. **分任务处理**：不要一次塞太多文件
4. **保存重要输出**：长分析结果建议保存到文件

### DON'T ❌

1. **不要发送敏感文件**：Gemini是云服务
2. **不要期望代码执行**：分析可以，执行找Claude
3. **不要忽略错误提示**：通常是路径或格式问题
