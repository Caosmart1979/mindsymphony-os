# Content Extraction & Information Hierarchy

将复杂内容转化为清晰PPT的核心方法论。

---

## 内容金字塔原则

```
                ┌─────────────┐
                │   核心结论   │  ← 1句话能说清
                └──────┬──────┘
                       │
              ┌────────┴────────┐
              │    支撑论点     │  ← 3-5个关键点
              │  (Key Messages) │
              └────────┬────────┘
                       │
        ┌──────────────┼──────────────┐
        │              │              │
   ┌────┴────┐   ┌────┴────┐   ┌────┴────┐
   │ 证据/数据 │   │ 证据/数据 │   │ 证据/数据 │
   └─────────┘   └─────────┘   └─────────┘
```

**应用规则**:
- 标题页 = 金字塔顶端
- 每个章节 = 一个支撑论点
- 每页内容 = 一个证据/数据点

## 内容压缩技术

### 学术内容压缩表

| 原始内容类型 | 压缩策略 | PPT呈现形式 | 压缩比 |
|-------------|---------|------------|--------|
| 摘要 (200-300字) | 提取核心发现 | 1-2个要点 | 10:1 |
| 背景/引言 (1-2页) | 只保留问题陈述 | 1页+1图 | 15:1 |
| 方法 (2-5页) | 流程图化 | 1-2页流程图 | 20:1 |
| 结果 (5-10页) | 图表+高亮数据 | 3-5页 | 3:1 |
| 讨论 (3-5页) | 提炼意义+局限 | 2页要点 | 10:1 |
| 参考文献 (50+条) | 省略或简化 | 0-1页 | ∞ |

### 文字精简公式

```
原始段落 → 核心句子 → 关键词组 → PPT文字

示例：
原始：本研究采用回顾性队列研究设计，纳入了2018年1月至2023年12月期间
在我院接受腹腔镜胃切除术的768例胃癌患者，按照是否合并糖尿病分为
糖尿病组(n=156)和非糖尿病组(n=612)...

核心句子：回顾性研究纳入768例腹腔镜胃切除患者，分为糖尿病组(156例)和
非糖尿病组(612例)

关键词组：腹腔镜胃切除 | 768例 | DM vs Non-DM (156 vs 612)

PPT呈现：
┌─────────────────────────────────┐
│  研究设计                        │
│  • 回顾性队列研究 (2018-2023)    │
│  • 腹腔镜胃切除术患者 (n=768)    │
│    - 糖尿病组: 156例            │
│    - 对照组: 612例              │
└─────────────────────────────────┘
```

## 信息层级设计

### 三层信息架构

```
Layer 1: 必看 (5秒内获取)
├─► 幻灯片标题
├─► 核心数字/结论
└─► 关键图表

Layer 2: 应看 (30秒内理解)
├─► 支撑要点 (3-5个)
├─► 数据对比
└─► 简要解释

Layer 3: 可看 (深入了解时)
├─► 详细数据
├─► 方法细节
└─► 参考来源
```

### 层级视觉映射

| 信息层级 | 字号范围 | 视觉处理 | 位置 |
|---------|---------|---------|------|
| Layer 1 | 28-40pt | 粗体、主色 | 顶部/中心 |
| Layer 2 | 18-24pt | 常规、次色 | 中部主体 |
| Layer 3 | 12-14pt | 浅色、小字 | 底部/脚注 |

## 数据呈现决策树

```
你有什么类型的数据？
│
├─► 单个关键数字
│   └─► 大字号突出显示 (Big Number Layout)
│       例: "42% 手术成功率提升"
│
├─► 趋势/变化
│   └─► 折线图 (Line Chart)
│       例: 术后并发症率随时间变化
│
├─► 类别比较
│   ├─► 2-4个类别 → 柱状图 (Bar Chart)
│   └─► 5+个类别 → 横向条形图
│
├─► 构成/占比
│   ├─► 2-5部分 → 饼图/环形图
│   └─► 6+部分 → 堆叠柱状图
│
├─► 两变量关系
│   └─► 散点图 (Scatter Plot)
│
├─► 多组对比
│   └─► 分组柱状图/表格
│
└─► 流程/关系
    └─► 流程图/关系图
```

## 每页内容检查表

### 内容密度标准

| 元素 | 最大数量 | 建议数量 |
|-----|---------|---------|
| 要点 | 5个 | 3个 |
| 图表 | 2个 | 1个 |
| 文字行数 | 8行 | 5行 |
| 总字数 | 75字 | 40字 |

### 单页自检

- [ ] 能用一句话说明这页的核心信息吗？
- [ ] 删掉任何元素会影响理解吗？（如果不会，删掉）
- [ ] 观众5秒内能抓住重点吗？
- [ ] 与上下页有清晰的逻辑连接吗？

## 学术论文 → PPT 映射模板

### 原始论文结构映射

```
论文章节              →    PPT页数    →    内容焦点
─────────────────────────────────────────────────────
Title/Authors         →    1页        →    标题+作者+机构
Abstract              →    0页        →    (不单独做，分布在各页)
Introduction          →    2-3页      →    研究背景+研究问题+假设
Methods               →    2-3页      →    研究设计+流程图+关键参数
Results               →    4-6页      →    主要发现+核心图表
Discussion            →    2页        →    意义+局限+未来方向
Conclusion            →    1页        →    核心结论 (可合并到最后)
References            →    0-1页      →    省略或致谢页
─────────────────────────────────────────────────────
总计                  →    12-17页
```

## 商业内容提炼

### 商业报告压缩原则

```
CEO/高管看什么？
│
├─► 结论和建议 (What)
│   "我们应该做X，预期收益Y"
│
├─► 关键支撑 (Why)
│   3个核心理由/数据点
│
└─► 下一步行动 (How)
    具体时间表和责任人
```

### SCQA框架

```
Situation (情境)  → 1页: 当前状态
Complication (冲突) → 1页: 面临的问题/挑战
Question (问题)   → (隐含在冲突中)
Answer (答案)     → 1-3页: 建议方案
```

## 常见错误与修正

| 错误模式 | 问题 | 修正方法 |
|---------|------|---------|
| 全文复制 | 把论文段落直接粘贴 | 提取关键词，重新组织 |
| 图表堆砌 | 每页放3+图表 | 每页1图表，配解读 |
| 信息平铺 | 所有要点同等重要 | 建立层级，突出核心 |
| 缺乏过渡 | 页与页之间跳跃 | 添加逻辑连接词/过渡页 |
| 细节过多 | 包含所有p值和CI | 只保留关键统计量 |
