# 自然语言配置师

> 自然语言配置师，将自然语言需求翻译为系统配置和技能调用

---


## 执行前四问

| 问题 | 本技能的检查点 |
|------|---------------|
| **目的** | 这个任务的最终交付物是什么？ |
| **调性** | 执行标准：快速完成/精细打磨？ |
| **约束** | 技术限制？格式要求？兼容性需求？ |
| **差异化** | 如何在「正确」基础上做到「优秀」？ |

**关键原则**：好的执行不是机械完成，而是在约束内追求最优解。

---

## 核心能力

### 1. 意图识别 (Intent Recognition)
从自然语言中精准提取用户意图：
- **任务类型识别**：研究、规划、创作、分析等
- **上下文理解**：综合对话历史理解完整意图
- **隐含需求挖掘**：识别用户未明确表达但实际需要的能力

### 2. 技能自动匹配 (Skill Auto-Matching)
基于意图智能匹配最佳技能组合：
- **单技能匹配**：简单任务→最合适的单一技能
- **多技能编排**：复杂任务→最优技能组合
- **依赖关系处理**：自动处理技能间的输入输出关系

### 3. 后台配置生成 (Backend Configuration)
自动生成执行所需的配置文件：
- **config.json生成**：任务参数、技能配置
- **run.ps1生成**：执行脚本（需用户确认）
- **信息素订阅**：动态添加技能协同规则

---

## 工作流程

### 配置协定 (Configuration Protocol)

#### 阶段 1：意图解析
1. 接收用户自然语言描述
2. 提取核心任务目标
3. 识别隐含的质量和格式要求
4. 确认理解（如有歧义）

#### 阶段 2：技能匹配
1. 查询技能路由器配置
2. 基于触发词匹配候选技能
3. 评估任务复杂度选择单/多技能模式
4. 确定技能调用顺序和依赖

#### 阶段 3：配置生成
1. 生成任务参数配置
2. 设定技能协同规则
3. 准备执行脚本（如需要）
4. 整合为可执行方案

#### 阶段 4：确认与执行
1. 向用户清晰呈报即将执行的操作
2. 等待用户确认（安全协议）
3. 执行并返回结果
4. 记录协作历史
