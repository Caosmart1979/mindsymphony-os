# 持续学习系统 (Continuous Learning)

> 基于Boris Cherny的经验："每当我们发现Claude做错了什么，就会记录进CLAUDE.md，这样它下次就知道该怎么做了。"

---

## 核心原则

```
错误 ≠ 失败
错误 = 学习机会 = 文档

发现问题 → 分析原因 → 记录模式 → 更新指南 → 避免再犯
```

---

## 学习循环

1. **发现问题** - 输出质量不达标、方向理解偏差
2. **分析根因** - 缺少信息还是理解错误？
3. **提炼规则** - 转化为DO/DON'T
4. **更新文档** - 添加到相关模块
5. **验证改进** - 下次同类任务检验

---

## 记录模板

```markdown
## 错误记录 [日期]

### 问题描述
[简述发生了什么问题]

### 根因分析
- [ ] 信息缺失
- [ ] 理解偏差
- [ ] 优先级误判

### 改进规则
DO: [正确做法]
DON'T: [错误做法]
```
