# 验证闭环系统 (Verification Loop)

> 基于Boris Cherny（Claude Code创始人）的核心洞见："给Claude一个验证工作的方法，质量提升2-3倍。"

---

## 核心原则

```
反馈闭环 = 质量保证

生成输出 → 验证检查 → 通过/失败 → 迭代修正（如需）
```

**关键洞见**：只要有反馈闭环，最终结果的质量会提升2-3倍。

---

## 验证方法矩阵

| 输出类型 | 验证方法 | 自动化程度 |
|----------|----------|-----------|
| **代码** | 运行测试/lint/typecheck | 高 |
| **文档** | 结构检查/一致性验证 | 中 |
| **计划** | 可行性评估/依赖检查 | 低 |
| **分析** | 数据源核对/逻辑自洽 | 中 |
| **创意** | 标准对照/用户测试 | 低 |
| **配置** | 语法验证/功能测试 | 高 |

---

## 工作流程

### 阶段 1：输出生成
1. 完成主要任务输出
2. 明确输出类型
3. 确定验证标准

### 阶段 2：验证方法选择
根据输出类型选择验证方法

### 阶段 3：执行验证
1. 运行选定的验证方法
2. 收集验证结果
3. 分析失败原因（如有）

### 阶段 4：决策与迭代
- 通过 → 输出交付
- 失败 → 分析原因 → 修正 → 重新验证（最多3轮）

---

## 验证模板

### 代码验证
```bash
bun run typecheck && bun run test && bun run lint
```

### 文档验证
- [ ] 标题层级正确
- [ ] 必需章节齐全
- [ ] 格式规范

### 计划验证
- [ ] 资源需求明确且可得
- [ ] 时间估算合理
- [ ] 有明确的成功标准
