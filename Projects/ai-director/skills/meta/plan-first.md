# 计划优先协议 (Plan-First Protocol)

> 基于Boris Cherny的经验："大多数会话从计划模式开始。先用计划模式和Claude来回沟通直到满意方案，再切换到执行。一份好的计划至关重要！"

---

## 核心原则

```
计划 > 执行

传统方式：用户请求 → 直接执行 → 发现问题 → 返工
计划优先：用户请求 → 输出计划 → 用户确认 → 执行 → 一次成功
```

---

## 触发条件

| 场景 | 计划必要性 |
|------|-----------|
| 复杂任务（步骤>3） | ⭐⭐⭐ 必须 |
| 创建PR | ⭐⭐⭐ 必须 |
| 系统设计 | ⭐⭐⭐ 必须 |
| 新功能 | ⭐⭐ 推荐 |
| 简单查询 | ⭐ 可跳过 |

---

## 计划模板

```markdown
## 计划确认

**任务**：[一句话描述]

**步骤**：
1. [Step 1] - [预期产出]
2. [Step 2] - [预期产出]
3. [Step 3] - [预期产出]

**风险**：[主要风险]

确认后开始执行？
```
